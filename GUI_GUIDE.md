# GUI 使用指南

## 启动 GUI

在项目根目录运行：

```bash
python run_gui.py
```

或者使用模块方式：

```bash
python -m pl_dss.gui
```

**注意：** 必须在项目根目录运行，不要直接运行 `python pl_dss/gui.py`

## 界面说明

### 输入区域

1. **未来14天的固定截止日期数量**
   - 使用上下箭头或直接输入数字
   - 范围：0-20
   - 示例：如果你有4个不可移动的截止日期，输入 4

2. **当前高负荷生活领域数量**
   - 使用上下箭头或直接输入数字
   - 范围：0-20
   - 高负荷领域包括：
     - 重大工作项目
     - 家庭危机
     - 健康问题
     - 搬家
     - 其他需要高认知负荷的事项

3. **最近3天的能量评分**
   - 为每一天评分（1-5分）
   - 1 = 精疲力竭
   - 5 = 精力充沛
   - 示例：如果最近三天感觉一般，可以输入 3, 3, 3

### 评估按钮

点击"评估状态"按钮后，系统会：
1. 验证输入数据
2. 计算当前状态
3. 显示详细结果

### 结果显示区域

结果包含以下信息：

1. **当前状态**（颜色编码）
   - 🟢 正常 (NORMAL) - 绿色
   - 🟠 压力 (STRESSED) - 橙色
   - 🔴 过载 (OVERLOADED) - 红色

2. **状态原因**
   - 解释为什么得出这个状态
   - 列出满足的条件

3. **激活的行为规则**
   - 当处于压力或过载状态时显示
   - 提供具体的行为约束建议

4. **恢复状态**
   - ✓ 已就绪：可以安全返回正常模式
   - ✗ 未就绪：列出阻塞恢复的条件

## 使用场景示例

### 场景1：正常状态
```
截止日期: 1
高负荷领域: 1
能量评分: 4, 5, 4
```
结果：正常状态，无需特殊约束

### 场景2：压力状态
```
截止日期: 3
高负荷领域: 1
能量评分: 4, 4, 3
```
结果：压力状态，建议谨慎接受新项目

### 场景3：过载状态
```
截止日期: 4
高负荷领域: 3
能量评分: 2, 3, 2
```
结果：过载状态，严格遵守降级规则

## 建议使用频率

**每周一次**（例如每周日晚上）进行评估，而不是每天使用。

## 自定义配置

如需修改阈值和规则，编辑项目根目录的 `config.yaml` 文件。

## 技术说明

- GUI 使用 Python 标准库 Tkinter 构建
- 无需额外安装 GUI 依赖
- 跨平台支持（Windows、macOS、Linux）
- 界面语言：中文
- 结果显示：中英文混合

## 故障排除

### 问题：GUI 无法启动

**解决方案：**
1. 确认 Python 版本 >= 3.8
2. 确认 Tkinter 已安装（通常随 Python 自带）
3. 在 macOS 上，确保使用系统 Python 或正确配置的 Python
4. **必须在项目根目录运行**

测试 Tkinter：
```bash
python -c "import tkinter; print('Tkinter OK')"
```

**正确的启动方式：**
```bash
# 在项目根目录
cd /path/to/life-planner
python run_gui.py
```

**错误的启动方式：**
```bash
# ❌ 不要这样做
python pl_dss/gui.py
```

### 问题：ModuleNotFoundError: No module named 'pl_dss'

**原因：** 没有在项目根目录运行，或使用了错误的启动方式。

**解决方案：**
```bash
# 确保在项目根目录
pwd  # 应该显示 .../life-planner

# 使用正确的启动方式
python run_gui.py
# 或
python -m pl_dss.gui
```

### 问题：配置文件错误

**解决方案：**
确保 `config.yaml` 文件存在且格式正确。

### 问题：输入验证失败

**解决方案：**
- 能量评分必须是 1-5 之间的整数
- 截止日期和领域数量必须是非负整数

## CLI vs GUI

| 特性 | CLI | GUI |
|------|-----|-----|
| 启动速度 | 快 | 稍慢 |
| 易用性 | 需要记住参数 | 直观 |
| 自动化 | 容易脚本化 | 不适合 |
| 视觉反馈 | 纯文本 | 颜色编码 |
| 适用场景 | 脚本、自动化 | 日常使用 |

根据你的需求选择合适的模式！
